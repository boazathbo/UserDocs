How to Configure SAML 2.0 for Front

This setup might fail without parameter values that are customized for your organization. Please use the Okta Administrator Dashboard to add an application and view the values that are specific for your organization.

  1. Login to your Front account as an administrator.

  2. Click Settings:

  3. Click Preferences, then select Single Sign On:

  4. Select SAML, then enter the following (see screen shot at end of step for reference):

    • ENTRY POINT: Copy and paste the following:

      Sign into the Okta Admin Dashboard to generate this variable.

    • Signing Certificate: Copy and paste the following (in PEM text format):

      Sign into the Okta Admin Dashboard to generate this variable.

    • Click Save.

  5. Make a copy of the [your-front-subdomain] value in the ENTITY ID field:

  6. In Okta, select the General tab for the Front app, then click Edit.

    • Paste the [your-front-subdomain] value you just copied into the Subdomain field.

    • Click Save.

  7. Done!


Notes:

SP-initiated flows and IDP-initiated flows are supported.

Just In Time (JIT) provisioning is not supported.
