How to Configure SAML 2.0 for Aspera Faspex V4

This setup might fail without parameter values that are customized for your organization. Please use the Okta Administrator Dashboard to add an application and view the values that are specific for your organization.

Follow the instructions below to configure SAML 2.0 for Aspera Faspex V4

  1. Contact your Aspera Faspex administrators, or follow the Aspera Faspex Admin Guide to enable SAML 2.0.

  2. Follow the instructions here to configure SAML integration with Okta on Aspera's side.

  3. Use the following IdP Metadata:

    Sign in to Okta Admin app to have this variable generated for you.

  4. If you are using multiple SAML providers, take note of the IdP ID you assigned to Okta.

  5. In Okta, select the General tab for the Aspera app, then click Edit.

    • Enter the LabelDomain or Base URL used by your org.

    • Choose the IdP ID you assigned to Okta. This will be "1" by default.

    • Click Save.

  6. The Application username format needs to be updated to use Okta username prefix. Do not assign users to this application until you update this setting, as follows:

    Still in Okta, select the Sign On tab, then click Edit.

    • Change the value of the Application username format to Okta username prefix.

    • Click Save.

  7. If necessary, you can add a filter for the group attributes. To do this, in Okta, select the Sign On tab for Aspera, then click on Edit.

    • Add a filter for the member_of group attribute. To include all groups in a filter, use Regex ".*" as shown on the screenshot below.

    • Click Save.

  8. Done!


Notes:

SP-initiated flows, IDP-initiated flows, and Just In Time (JIT) provisioning are all supported.
