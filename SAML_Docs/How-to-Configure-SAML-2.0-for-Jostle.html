How to Configure SAML 2.0 for Jostle

This setup might fail without parameter values that are customized for your organization. Please use the Okta Administrator Dashboard to add an application and view the values that are specific for your organization.

Note:

There are two OAN Jostle apps:

  • Use the Jostle app if you do not have a custom Jostle subdomain and your login URL looks like this: https://[platform|us|ca|eu|ap].jostle.us/jostle-prod/.

  • Use the Jostle with Subdomain app if you have a custom Jostle subdomain and your login URL looks like this: https://[yourSubDomain].jostle.us/jostle-prod/.

Configuring SAML 2.0 for Jostle

  1. Copy the following IDP Metadata into a text file. Save it with the file name metadata.xml.

    Sign into the Okta Admin dashboard to generate this value.

  2. Send a message to Jostle support at support@jostle.me and request SAML support. Include the following two items in the message.

    1. Attach the metadata.xml file you saved in step 1.

    2. Copy and paste the following text into the message and label it IDP Issuer/Entity ID.

      Sign in to the Okta Admin app to generate this variable.
  3. After Jostle confirms that SAML is enabled, sign in to Jostle as an administrator.
  4. Follow the instructions in the Jostle document Converting Password Authenticated Users to SSO using either or both methods described in that document.
  5. Done!

    Note: Jostle supports both IdP-initiated and SP-initiated flows. Just in Time (JIT) provisioning is not supported.
