How to Configure SAML 2.0 for Teem

This setup might fail without parameter values that are customized for your organization. Please use the Okta Administrator Dashboard to add an application and view the values that are specific for your organization.

  5. Teem will process your request. When you receive a confirmation that SAML is enabled, you can start assigning users to the application.

  6. Done!

Notes:

SP-initiated flows and Just In Time (JIT) provisioning are supported.

IdP-initiated flows are not supported.

For the convenience of your end users, Okta recommends that you hide the Teem app and set up a Bookmark app with the Teem logo. For instructions on setting up a Bookmark app and hiding the original app, see Simulating an IDP-initiated Flow with the Bookmark App.

The link for an SP-initiated flow is: https://[yourSubDomain].eventboard.io/login/, where [yourSubDomain] is the domain you have already set up.
