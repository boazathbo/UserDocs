How to Configure SAML 2.0 for MobileIron

This setup might fail without parameter values that are customized for your organization. Please use the Okta Administrator Dashboard to add an application and view the values that are specific for your organization.

  1. Login to MobileIron as an administrator.

  2. Navigate to Admin > Identity.

  3. Click the Set Up An Identity Provider button:

  4. Click the Generate Key button:

  5. Save the Key and Host values.

  6. Copy the following IDP Metadata and save it as metadata.xml.

    Sign in to Okta Admin app to have this variable generated for you.

  7. Click Choose File and upload the metadata.xml file you just saved.

  8. Click Done.

  9. In Okta, select the General tab for the MobileIron app, then click Edit.

    • Enter the Key and Host values you saved earlier into the corresponding fields.

    • Click Save.

    • Optional if you need to send the Groups as part of the SAML assertion: Select the Sign On tab, then click Edit.

    • Select your preferred Group filter from the dropdown list (the Regex rule with the value ".*" in order to send *all* groups to the MobileIron instance we used in our example). .

    • Click Save.

  10. Done!


Notes:

SP-initiated flows and IDP-initiated flows are supported.

Just In Time (JIT) provisioning is not supported.

For SP-inititated Flows:

  1. Open the login URL.

  2. Enter your Username.

  3. Click Sign In.
